## What I Actually Do

Most engineers can call an API and call it "AI." I go deeper:

- **Browser-Native ML** — running inference client-side with WebGL + MediaPipe. No backend, no data leaving the user's device, near-zero latency
- **LLM Integration** — streaming interfaces, tool/function calling, structured outputs, RAG pipelines, prompt engineering that holds up in production
- **AI Product Thinking** — I care about what the AI *does for the user*, not just whether it works technically

I've built and shipped products solo. I don't need hand-holding to go from idea → deployed.
